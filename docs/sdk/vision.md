# Visão

## Objetivo

Preservação tecnológica ativa.
RetroNet SDK não tenta substituir tecnologias antigas. A proposta é reaproveitar o que ainda tem valor e devolver utilidade prática a softwares, protocolos e ferramentas retro.

## Princípios

- compatibilidade acima da modernização
- suporte a máquinas limitadas
- suporte a redes lentas
- protocolo estável e extensível
- SDK como plataforma, não apenas cliente
- integração com DOS, FreeDOS, Windows 3.x, Windows 9x, Linux, emuladores e hardware real

## Plataformas alvo

- MS-DOS
- FreeDOS
- Windows 3.x
- Windows 9x
- Windows 2000
- Linux
- DOSBox-X
- PCem
- 86Box
- QEMU
- hardware real

## Status

Esta é a visão do SDK. A implementação será incremental e orientada por compatibilidade retro.

## Uso futuro

O SDK deverá servir como base para:

- clientes de mensagens
- TSRs
- interfaces texto e VGA
- integração com QWK
- ponte para redes legadas

- definir o núcleo `librn`
- separar interfaces por capacidade
- documentar bindings por linguagem
- estabilizar o contrato de transporte

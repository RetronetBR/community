# Contributing

RetroNet é um projeto orientado a comunidade e compatibilidade retro.

## Regras gerais

- manter compatibilidade retro
- evitar dependências externas
- evitar frameworks pesados
- simplicidade acima de complexidade
- pensar em DOS, Z80 e 8088 antes de adicionar recursos
- não quebrar compatibilidade existente

## Prioridades

- texto
- arquivos simples
- portabilidade
- baixo consumo

## O que preservar

- o fluxo store-and-forward
- os arquivos texto como base do runtime
- a separação entre cliente, nó e peer
